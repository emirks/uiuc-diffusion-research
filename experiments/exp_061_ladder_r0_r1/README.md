# exp_061 — eval-ladder rungs R0/R1: base-prior and conditioning-suppression baselines

## Question

Rungs 0 and 1 of the eval ladder, zero training, paired by item and seed:

- **R0** — base LTX-2 19B dev, **prompt only** (type-blind endpoint caption),
  NO conditioning, NO adapter: does the base prior contain each transition
  effect at all?
- **R1** — same base model + **endpoint conditioning** (prefix 9f + suffix 8f
  from the item clip, exp_051 recipe), prompt only, NO reference, NO adapter:
  how much does endpoint conditioning suppress (or enable) the effect relative
  to R0?

## Setup

- **Base checkpoint**: `cache/huggingface/ltx2_models/ltx-2-19b-dev.safetensors`
  + Gemma-3-12B QAT-q4 text encoder — the exact base the exp_056 IC-LoRA arm
  used (exp_056 `config_ic.yaml` / exp_060 `config.yaml`). No LoRA loaded.
- **Recipe** (exp_057/060 contract verbatim, via the trainer's
  `ValidationRunner`): 480×640×121 @ 24 fps, 30 steps, CFG 4.0, STG 1.0
  `stg_v` blocks [29], negative prompt "worst quality, inconsistent motion,
  distorted, jittery". R1 conditions cut by `make_conds.py` (exp_060 ffmpeg
  recipe: first-9 / last-9 frames, suffix consumed as `num_frames=8` —
  exp_051 causal-VAE rule). R0 = identical `ValidationSample` with
  `conditions=[]` (plain T2V), so the pair differs ONLY in conditioning.
- **Prompts**: `ICTRANS ` + type-blind endpoint caption of the item clip —
  identical string in both arms. Captions reused from exp_056/057/058/060
  caption files; 4 missing ones generated by `caption_missing.py` (exact
  exp_058/060 Gemini captioner, temp 0). The `ICTRANS` trigger is kept for
  prompt parity with the exp_056/057 base twins and future adapter rungs;
  exp_051 showed the token is inert without the adapter.
- **Items** (`select_items.py`, deterministic, from **frozen split v1
  post-audit** — `data/processed/transitions_std121/split_v1.json`; audit
  `split_v1_audit.md` flagged 12 test picks in 4 classes, remediated
  mechanically, live_concert went all-train):
  - every class with test clips → its test clips (11 classes × 2 + 17 × 1 = 39),
  - all-train classes (10 with n < 4 + live_concert by remediation) → the
    class's certified bar-pair endpoints clip (`sib__<class>` clip A from
    `outputs/eval/certification/3.0.0-draft.8/manifests/siblings.json`,
    the exp_060 pattern; 9 classes),
  - the two n = 1 singletons (jump_transition, seamless_transition, no bar
    pair) → their single clip.
  → **50 items covering all 39 classes**.
- **Seeds**: 42, 43, 44 — same items AND same seeds in both arms
  (paired design, non-negotiable). Total 50 × 3 × 2 = **300 videos** (cap 500).
- **Budget math**: exp_060 measured 60 reference-conditioned videos
  (9600-token sequences) in 2:01 h on one H100 ≈ 121 s/video including model
  load. R0/R1 run 4800-token sequences (no reference) → ≤ ~110 s/video is
  conservative. 300 × 110 s ≈ 9.2 GPU-hours; 4 array tasks (75 videos,
  ≤ 2.5 h each) at ≤ 2 concurrent GPUs ≈ 5–6 h wall-clock — well inside the
  14 h overnight window on `HCESC-H100-normal`. No de-scoping needed.

## Expected outcome

Pre-registered: (a) R0 should produce prompt-plausible content but mostly
generic scene changes / crossfades — per-class presence of the actual effect
morphology is the readout (exp_051's base arm showed generic smoke-curtain
filling); (b) R1 should anchor endpoints (mechanism-robust per exp_051 even
without a matched adapter) and is expected to SUPPRESS mid-clip effect
invention relative to R0 (conditioning pins the trajectory ends; the base
prior has no incentive to insert the effect); (c) any class where R0 already
shows the effect marks the base prior as containing it — adapter rungs must
beat R1, not R0, there. **Scoring is deliberately NOT run** — blocked until
the sidedness re-annotation is validated; `eval_manifest_r0/r1.json` rows use
`reference_video` = the item's own ground-truth clip (no demo exists in these
arms) and are ready for score.py once unblocked.
