# Certification records

One committed markdown record per certified harness version (`v3.0.md`, …),
written by the §6 certification run and required before the matching
`eval/vX.Y` git tag may be created. Each record contains: the pre-registered
bars (frozen before the run), the measured numbers and verdicts, the corpus
manifest hash, σ_seed, and pointers to the full artifacts under
`outputs/eval/certification/<version>/`.

No record here yet: `3.0.0-draft.*` versions are by definition uncertified.
The first record lands when the health-assessment phase (SPEC §0: O3, O4)
executes against the locked v3 instrument.
