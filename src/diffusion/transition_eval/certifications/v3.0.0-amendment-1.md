## 2. τ_copy pinned at 0.858 (owner decision, 2026-07-14)

The certified bar-4 rule pins τ_copy at the honest/splice gap midpoint; the
certification run calibrated that midpoint to **0.858** (splice min 0.914,
honest max 0.802, gap 0.112). The owner adopted 0.858 as the operating
threshold.

Implementation note — why this needs no re-run and no code change: `copy_max`
is stored raw in every row; τ enters only when converting it to the
`near_copy` flag. The scoring code's constant (`m2_integrity.TAU_COPY =
0.88`, marked DRAFT pending this calibration) is a per-row default; analysis
and reports MUST recompute the flag as `copy_max >= 0.858` and cite this
amendment. Rows' embedded `near_copy` therefore reflects the draft 0.88 until
a future version bumps the constant (a code change that, per §7, rides with
the next re-certification — it must not be edited at the tag).

The split-audit near-duplicate scan and all ladder reporting use 0.858.
